# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in the Agent Receipts documentation site, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, email **otto@0tt0.net** with:

- A description of the vulnerability
- The impact you believe it has
- Any suggested mitigations

You should receive an acknowledgement within 48 hours. We will work with you to understand the issue and coordinate a fix before any public disclosure.

## Scope

The following are in scope for security reports:

- Vulnerabilities in the site build or deployment pipeline
- Exposed secrets or credentials
- Content that misrepresents the protocol's security properties

For issues with the protocol itself, SDKs, or schemas, report to the relevant repository:

- [agent-receipts/spec](https://github.com/agent-receipts/spec/security)
- [agent-receipts/sdk-ts](https://github.com/agent-receipts/sdk-ts/security)
- [agent-receipts/sdk-py](https://github.com/agent-receipts/sdk-py/security)

## Disclosure Policy

- We aim to confirm and triage reports within 48 hours.
- We will coordinate a fix and release timeline with the reporter.
- We credit reporters in the release notes unless they prefer to remain anonymous.
- We ask that you give us reasonable time to address the issue before public disclosure.
