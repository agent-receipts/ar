## Install

### Homebrew (macOS, Linux)

```bash
brew install agent-receipts/tap/mcp-proxy
```

### Prebuilt binaries

Download a tarball for your platform (darwin/linux, amd64/arm64) from the [releases page](https://github.com/agent-receipts/ar/releases?q=mcp-proxy) and move `mcp-proxy` onto your `$PATH`.

### From source

```bash
go install github.com/agent-receipts/ar/mcp-proxy/cmd/mcp-proxy@latest
```

Requires Go 1.26+. No CGO or external C libraries â€” the proxy uses pure Go SQLite.

## Verify installation

```bash
mcp-proxy -version
```

If `mcp-proxy` isn't found after `go install` (bash/zsh say `command not found`; fish says `Unknown command`), the Go bin directory isn't on your `$PATH`. `go install` writes binaries to `$GOBIN` when it's set, otherwise to `$(go env GOPATH)/bin` (usually `~/go/bin`). Add whichever applies to your shell profile:

```bash
# bash / zsh
export PATH="$(go env GOPATH)/bin:$PATH"
```

```fish
# fish
fish_add_path (go env GOPATH)/bin
```

## Basic usage

Wrap any MCP server by passing its command as arguments. For a quick smoke test, use the official filesystem MCP server (requires Node.js/`npx` â€” if you don't have it, substitute any MCP server command you already have):

```bash
mcp-proxy npx -y @modelcontextprotocol/server-filesystem ~/Documents
```

The proxy intercepts stdin/stdout, logs every tool call, and forwards messages transparently. By default it generates an ephemeral Ed25519 key pair for signing receipts. Press `Ctrl-C` to stop.

For wiring the proxy into an actual client, jump to [Claude Code](/mcp-proxy/claude-code/), [Claude Desktop](/mcp-proxy/claude-desktop/), or [Codex](/mcp-proxy/codex/).

## Approval server (required with the default policy)

The default policy contains a `pause` rule (`pause_high_risk`, `min_risk_score: 50`). Any tool call above that score is held until an approver POSTs a decision. **Leave this unconfigured and every high-risk write â€” `create_pull_request`, `merge_pull_request`, destructive SQL, etc. â€” will fail with `-32002: approval timeout or rejected`** after `-approval-timeout` elapses. Lower-risk calls (reads, `push_files`, `create_branch`) still succeed, which makes the failure look tool-specific.

**Minimum viable setup:** pin a port with `-http 127.0.0.1:8081` and run an approver against it. The dedicated [Approval Server](/mcp-proxy/approval-ui/) page documents the real HTTP routes (`POST /api/tool-calls/{id}/approve` and `/deny` â€” there is no index page at `/`) and shows a minimal shell approver.

**If you don't want human-in-the-loop review**, opt out explicitly by loading a `-rules` YAML that contains no `pause` or `block` rules. The [Approval Server page](/mcp-proxy/approval-ui/#opting-out-of-approval) has templates. Do not leave the default policy active without an approver â€” that state is a misconfiguration, not a mode of operation.

## Persistent signing key

Generate a key pair and use it across sessions:

```bash
# Generate a key pair (any Ed25519 PEM tool works)
openssl genpkey -algorithm Ed25519 -out private.pem
openssl pkey -in private.pem -pubout -out public.pem

# Run with persistent key
mcp-proxy -key private.pem npx -y @modelcontextprotocol/server-filesystem ~/Documents
```
