# Retrospective — Orchestrated Issue Batch (2026-05-31)

Internal operations retrospective on an experiment: farming a multi-wave backlog
to a single **orchestrator** agent that spawns isolated sub-agents per issue,
drives the Copilot review loop, and gates every merge through a human.

This is process state, not a deliverable.

## 1. The experiment

Four planned waves, by component:

1. **docs/prose** — #73, #441, #536 (+ follow-up), #631
2. **mcp-proxy** — #213 → #214 → #174 → #172 → #153
3. **SDK** — #480 → #534
4. **CI gates** (human-gated) — #653 (+#654) → #655 → #656

Mechanics: one orchestrator; one issue → one git worktree → one branch → one PR;
read-only "currency audits" before implementing; human merge gate on everything.

## 2. Outcome

**Headline: 2 of the 3 audited waves were already implemented — only the GitHub
issues were stale-open.** The batch's real value was *triage*, not *build*.

| Wave | Issues | Result |
|------|--------|--------|
| 1 — docs | #73 / #441 / #536 / #536-fu / #631 | PRs #677–#681 merged |
| 2 — mcp-proxy | #213 / #214 / #174 / #172 / #153 | #213 (#682) & #172 (#690) merged; **#214 / #174 / #153 closed as already-done** |
| 2 — follow-ups | #691 / #692 | PRs #693 / #694 merged |
| 3 — SDK | #480 / #534 | **both closed as already-done** (shipped earlier in #565 / #611 / #663) |
| 4 — CI gates | #653 / #654 / #655 / #656 | #653 (#696) & #654 (#698) merged; **#656 (#699) open**; **#655 deferred** (needs protocol-version design) |

Ledger: ~8 PRs merged · 5 issues closed as already-done · 1 PR open · 2 follow-ups
filed (#691 / #692) · 1 deferred (#655) · 1 process issue filed (#700, lint `scripts/`).

## 3. What worked

- **Audit-before-build.** Cheap read-only currency checks (~40–60K tokens each)
  collapsed Waves 2 and 3; they saved an estimated ~7 author cycles. Single
  highest-ROI move of the batch.
- **One issue → one worktree → one PR.** Clean isolation; failures stayed local.
- **Human merge gate on every CI/workflow change** — never auto-merged.
- **Halt-to-human on ambiguity and at caps** (e.g. #680 least-privilege nit,
  #655 design dependency).
- **A "short-leash" policy** (adopted mid-batch) curbed the per-round nit chase.

## 4. What cost time / went wrong

- **The Copilot review loop was the dominant time-sink** (see §5). Re-reviews on
  every push, 16 comments *after* merge on #698, ~9 zero-information advisory
  comments. Bimodal: valuable on prose, mostly noise on already-clean code.
- **Lint gap (#700).** The release-gate helpers under `scripts/` (~1.5k lines of
  Python) are under no linter, so an unused import reached review.
- **Orchestrator reliability errors.** A PR body was overwritten with a
  reconstruction made *before* the real body was fetched; state was assumed
  rather than fetched more than once. Caught and corrected, but trust-relevant.
- **Author-agent quality tail.** The largest author run (~205K tokens / 374 tool
  calls / ~22 min, for #656) shipped a README with a wrong command — long-run
  degradation.
- **Environment / tooling friction (recurring):**
  - Commit-signing fails from worktrees outside the project tree (`/tmp`); it
    works from in-repo worktrees (`.claude/worktrees/`).
  - `lefthook` not on `PATH` in some shells → pre-commit/commit-msg/pre-push
    hooks silently skipped.
  - The parallel-worktree race churned the orchestrator's own working tree on
    *every* sub-agent launch, requiring a `git checkout -f` reset each time.
  - A "Generated by …" trailer was auto-appended to *every* PR body and had to be
    stripped manually.

## 5. Copilot review: noise-to-findings analysis

Classifying ~45 Copilot comments across PRs #680 / #690 / #696 / #698 / #699
(excludes the third-party lint bot):

| Category | ~count | share |
|----------|-------:|------:|
| Genuine bugs fixed (real correctness) | ~8 | ~18% |
| Real but out-of-scope → filed as follow-ups (#691 / #692) | ~3 | ~7% |
| Minor-but-valid nits (doc / comment / test-name / style) | ~13 | ~29% |
| Advisory boilerplate ("workflow needs maintainer review") | ~9 | ~20% |
| Pure duplicates (same comment reposted) | ~8 | ~18% |

**~24% of comments changed shipped code in a way that mattered; ~76% was nits,
boilerplate, or duplicates.**

Qualitative findings:

- **Bimodal value.** On the docs PR (#680) Copilot was genuinely useful — it found
  ~6 real recipe bugs (cross-shell `wait` race, GOPATH/`PATH`, empty-chain guard,
  "recipe never emits", `if: always()` skip-on-failure, toolchain pin). On the
  polished CI-gate PRs it degraded to nits + boilerplate + duplicates (#698
  post-merge alone: 16 comments, **0 live bugs**).
- **The one mechanically-catchable defect — the unused import — Copilot missed**;
  a separate static-analysis bot caught it.
- **"Copilot-clean" is not a terminal state.** It re-reviewed on every push *and
  after merge*; the "needs maintainer review" advisory re-fired on every revision.
- **Cost shape:** ~5 PRs × 2–4 rounds, each round = async wait + a fix cycle +
  re-trigger, with steeply diminishing returns per round.

**Recommendation:** treat Copilot as **advisory and front-loaded** — run a self
code-review before the *first* request, act on round 1, and stop gating after.
It earned its keep on prose; it was a net time-sink on already-clean code.

## 6. Orchestration model: single vs one-per-wave

What ran: one orchestrator, waves handled largely serially, one sub-agent per issue.

Verdict for this batch: **single was about right — but partly by luck of the
result.**

- The highest-leverage move (audit-before-build) is best applied **once,
  consistently**; a single scout did it cheaply. Four parallel orchestrators would
  each have re-derived it.
- Only **1 of 4 waves (CI gates) had real implementation work.** Parallel
  orchestrators would mostly have parallelized "discover it's already done" —
  modest gain, against 4× the human-interface surface, duplicated environment
  setup, and no shared learning (each would re-hit the signing / hook / trailer
  quirks independently).
- The real bottleneck was **wall-clock on the Copilot loops**, which *would*
  parallelize across orchestrators — but only matters when multiple waves carry
  substantive PRs.

**Recommended model:** one **scout orchestrator** runs the cross-cutting audit +
triage/close of stale issues for *all* waves; **then** fan out **per-wave
orchestrators in parallel only for waves with genuine, independent
implementation.** Within a wave, shared files force serialization regardless (all
three Wave-4 gates edit `release-sdk-*.yml`). Per-wave orchestration pays off
specifically when ≥2 waves have real, independent code.

## 7. Self-review vs mechanical lint

Self-review is already mandated three ways in `AGENTS.md` ("Self-review before
committing"; "Re-read your full diff"; "Run language-specific linters"). The
agents did self-review. The unused import still shipped because **self-review is a
judgment step and an unused import is exactly the class of defect that human/agent
eyes skim past but a linter catches deterministically** — and "run ruff" silently
no-ops for `scripts/` (no config there). The fix is mechanical enforcement (#700),
not more instructions.

## 9. Cost

~1.12M sub-agent tokens across ~13 reported agents — a **lower bound**: it excludes
the orchestrator's own usage (large: every webhook, PR read, and the full
`AGENTS.md` re-injected on each worktree visit) and any agents summarized out of
context. Realistic all-in is meaningfully higher.

| Agent | tokens | | Agent | tokens |
|-------|-------:|-|-------|-------:|
| #653 gate | 142K | | #172 shutdown | 133K |
| #654 gate | 112K | | #690 fix | 70K |
| #656 gate | **205K** | | #691 | 86K |
| #680 emitter | 71K | | #692 | 61K |
| mcp-proxy audit | 63K | | #214 | 37K |
| #680 recipe | 55K | | Wave-3 audit | 47K |
| Wave-4 audit | 39K | | | |

Shape: **audits were cheap and high-ROI**; **gate authors were the expensive tail**,
with #656 an outlier (205K / 374 tool calls / ~22 min) — and the run that shipped
the README defect, suggesting long-run degradation. Splitting/checkpointing long
authoring tasks is worth trying.

## 10. Recommendations

1. **Lint `scripts/`** (#700) — F-class enforcement pre-merge.
2. **SessionStart hook** to fix environment up front: `git remote set-head origin
   main`, SDK installs (`pnpm install`, `uv sync --all-extras`), `cargo install
   convco`, `lefthook install`.
3. **Treat Copilot as advisory, front-loaded** — self code-review before the first
   request; act on round 1; do not chase post-merge or duplicate rounds.
4. **Build PR bodies only from the fetched body**; strip the auto-trailer as a
   standard step (or suppress it).
5. **Reset the orchestrator worktree after every sub-agent launch** (became a
   reflex; make it explicit).
6. **Cap single-agent runtime/turns**; split or checkpoint long gate work.
7. **Orchestration:** scout for audit + triage across all waves; parallel per-wave
   orchestrators only for waves with real, independent implementation.
