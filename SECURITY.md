# Security Policy

## Reporting a Vulnerability

The agent-receipts team takes security seriously. We appreciate your efforts to responsibly disclose any security vulnerabilities you find.

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report them via [GitHub Security Advisories](https://github.com/agent-receipts/sdk-go/security/advisories/new).

Include as much of the following information as possible:

- A description of the vulnerability
- Steps to reproduce the issue
- Potential impact of the vulnerability
- Any suggested fixes (if applicable)

## Response Timeline

- **Acknowledgement**: We will acknowledge receipt of your report within 48 hours.
- **Assessment**: We aim to provide an initial assessment within 5 business days.
- **Resolution**: We will work with you to understand and resolve the issue promptly.

## Disclosure Policy

- We follow a coordinated disclosure process.
- We will credit reporters (unless anonymity is requested) when publishing fixes.
- We ask that you allow us reasonable time to address the issue before any public disclosure.

## Scope

This policy applies to the `sdk-go` repository and any related packages published under the `agent-receipts` GitHub organization.
